# TimeCanvas 画面設計書（Phase 1）

> [requirements.md](requirements.md) の Phase 1 機能要件に対応する画面設計。
> デザインの方向性：**Outlook カレンダー風**（学習コストゼロ）。

├──────────────┬─────────────────────────────────────────────────┤
│              │      月 29   火 30   水 1    木 2    金 3  ...   │
│  ミニカレンダー │ 8:00 ┌──────┐                                    │
│  ┌──────────┐ │      │開発  │        ┌──────┐                   │
│  │ 2026年7月 │ │ 9:00 │      │        │会議  │                   │
│  │ … 2 3 4 …│ │      └──────┘        └──────┘                   │
│  └──────────┘ │10:00          ┌──────┐         ━━━━ ← 現在時刻線  │
│              │      …週カレンダーグリッド（30分刻み）…             │
│  カテゴリ     │                                                  │

### 構成要素

| 要素 | 説明 |
|------|------|
| ナビゲーションレール | ウィンドウ最左端の縦型アイコンバー（幅 48px）。カレンダー / タスク / 分析 / ノート を切替。下端に設定。Outlook のアプリバーと同じ位置づけ |
| ツールバー | 今日ボタン / 週送り / 期間表示 / ビュー切替（日・週・月）/ 設定 |
| サイドパネル | ミニカレンダー（月）、カテゴリ一覧（表示フィルタ兼用）、週サマリー |
| カレンダーグリッド | 7 列 × 時間軸。30 分ごとの罫線、15 分スナップ |
| 現在時刻ライン | 今日の列に赤系の水平線（Outlook 同様） |
| エントリブロック | カテゴリ色の背景 + 左端に濃い色のバー。タイトルと時刻を表示 |

## 2. 操作フロー

### 2.1 エントリ作成（メインの操作）

1. グリッド上をドラッグして時間帯を選択
2. その場に**クイック作成ポップオーバー**が出る：タイトル入力 + カテゴリ選択 + [保存]
3. Enter で保存。「詳細を編集」リンクで詳細ダイアログへ

### 2.2 エントリ編集

- **ドラッグ移動**：ブロックを掴んで別の時間帯・曜日へ
- **伸縮**：上下端のハンドルをドラッグ
- **クリック**：概要ポップオーバー（時刻・カテゴリ・メモ冒頭 + 編集/削除ボタン）
- **ダブルクリック**：詳細編集ダイアログを直接開く

### 2.3 詳細編集ダイアログ

```
┌──────────────────────────────┐
│ タイトル [________________]   │
│ カテゴリ [■ 開発        ▼]   │
│ 開始 [2026/07/02] [09:00 ▼]  │
│ 終了 [2026/07/02] [10:30 ▼]  │
│ メモ                          │
│ [________________________]   │
│ [________________________]   │
│          [削除]  [キャンセル] [保存] │
└──────────────────────────────┘
```

## 3. ビジュアルデザイン

### 3.1 カラーパレット（Outlook 風・ライトテーマ）

| 用途 | 色 |
|------|-----|
| アクセント（選択・今日・ボタン） | `#2564CF`（Outlook ブルー系） |
| 背景 | `#FFFFFF` / サイドパネル `#FAF9F8` |
| 罫線 | `#EDEBE9`（30分線は更に淡く） |
| テキスト | `#252423` / 補助 `#605E5C` |
| 現在時刻ライン | `#D13438` |

カテゴリ既定色（Outlook の分類色に準拠）：
青 `#2564CF` / 緑 `#498205` / 紫 `#8764B8` / オレンジ `#CA5010` / 赤 `#D13438` / ティール `#038387`

### 3.2 タイポグラフィ

- フォント：`Segoe UI, Yu Gothic UI, sans-serif`（Windows 標準）
- ベース 14px、時刻軸ラベル 12px、ツールバー期間表示 20px semibold

### 3.3 エントリブロックのスタイル

- 背景：カテゴリ色の淡色（例：カテゴリ色 15% 不透明度）
- 左端：カテゴリ色の 4px バー
- 角丸 4px、ホバーで薄いシャドウ
- 選択中：カテゴリ色のボーダー 2px

## 4. ナビゲーションレール（画面切替）

ウィンドウ最左端に幅 48px の縦型アイコンバーを常設する。

```
┌──┬──────────────────────────────┐
│📅│ ← カレンダー（Phase 1・既定）    │
│☑ │ ← タスク（Phase 2）             │
│📊│ ← 分析（Phase 3）               │
│📓│ ← ノート（Phase 4）             │
│  │                               │
│⚙ │ ← 設定（下端固定）              │
└──┴──────────────────────────────┘
```

- 選択中のアイコンはアクセント色の背景 + 左端に 3px のアクセントバー
- 未実装フェーズのアイコンは Phase 1 では非活性（グレー + ツールチップ「近日対応」）
- 各画面はレール以外の領域（サイドパネル + コンテンツ）を丸ごと入れ替える

## 5. 設定画面

タブ構成のダイアログ：

1. **カテゴリ管理** — 一覧、追加、名前/色編集、アーカイブ
2. **表示** — 週の開始曜日（月/日）、表示時間帯（既定 7:00-22:00 にスクロール）、時刻刻み
3. **データ** — バックアップ先フォルダの指定（OneDrive 推奨）、今すぐバックアップ、バックアップから復元、エクスポート（JSON / CSV）、データ保存先の表示

## 6. 将来の拡張とレイアウトの関係（参考）

- Phase 2（タスク）：左＝タスクリスト（見積時間付き）、右＝ミニ週カレンダー。タスクをカレンダーへドラッグして実績化
- Phase 2（検索）：ツールバー右上に検索ボックス（`Ctrl+F`）。入力でコンテンツ領域が結果リストに切り替わり、項目クリックで該当週へジャンプ + ハイライト（仕様詳細は requirements.md 9.1）
- Phase 3（分析）：期間セレクタ + カテゴリ別集計、見積 vs 実績、年間ヒートマップ（GitHub の草スタイル）
- Phase 4（ノート）：左＝フォルダツリー、右＝Markdown エディタ/プレビュー

ナビゲーションレール + 「サイドパネル + コンテンツ」の 2 段構造は、全画面がこの型に収まるように設計している。
