# TimeCanvas 画面設計書（Phase 1）

> [requirements.md](requirements.md) の Phase 1 機能要件に対応する画面設計。
> デザインの方向性：**Outlook カレンダー風**（学習コストゼロ）。

### 構成要素

| 要素 | 説明 |
|------|------|
| ナビゲーションレール | ウィンドウ最左端の縦型アイコンバー（幅 48px）。カレンダー / タスク / 分析 / ノート を切替。下端に設定。Outlook のアプリバーと同じ位置づけ |
| ツールバー | 今日ボタン / 週送り / 期間表示 / ビュー切替（日・週・月）/ 設定 |
| サイドパネル | ミニカレンダー（月）、カテゴリ一覧（表示フィルタ兼用）、週サマリー |
| カレンダーグリッド | 7 列 × 時間軸。正時=実線・30 分=破線の罫線、15 分スナップ。1 列 150px・1 時間 64px の固定寸法（ウィンドウリサイズで伸縮せず横スクロール。時刻ガターとヘッダは追従固定） |
| 現在時刻ライン | 今日の列に赤系の水平線（Outlook 同様） |
| エントリブロック | カテゴリ色の背景 + 左端に濃い色のバー。タイトルと時刻を表示 |

## 2. 操作フロー

### 2.1 エントリ作成（メインの操作）

1. グリッド上をドラッグして時間帯を選択
2. その場に**クイック作成ポップオーバー**が出る：タイトル入力 + カテゴリ選択 + [保存]
3. Enter で保存。「詳細を編集」リンクで詳細ダイアログへ

### 2.2 エントリ編集

- **ドラッグ移動**：ブロックを掴んで別の時間帯・曜日へ
- **伸縮**：上下端のハンドルをドラッグ
- **クリック**：概要ポップオーバー（時刻・カテゴリ・メモ冒頭 + 編集/削除ボタン）
- **ダブルクリック**：詳細編集ダイアログを直接開く

### 2.3 詳細編集ダイアログ

```
┌──────────────────────────────┐
│ タイトル [________________]   │
│ カテゴリ [■ 開発        ▼]   │
│ 開始 [2026/07/02] [09:00 ▼]  │
│ 終了 [2026/07/02] [10:30 ▼]  │
│ メモ                          │
│ [________________________]   │
│ [________________________]   │
│          [削除]  [キャンセル] [保存] │
└──────────────────────────────┘
```

## 3. ビジュアルデザイン

### 3.1 カラーパレット（Outlook 風・ライトテーマ）

| 用途 | 色 |
|------|-----|
| アクセント（選択・今日・ボタン） | `#2564CF`（Outlook ブルー系） |
| 背景 | `#FFFFFF` / サイドパネル `#FAF9F8` |
| 罫線 | `#EDEBE9`（30分線は更に淡く） |
| テキスト | `#252423` / 補助 `#605E5C` |
| 現在時刻ライン | `#D13438` |

カテゴリ既定色（Outlook の分類色に準拠）：
青 `#2564CF` / 緑 `#498205` / 紫 `#8764B8` / オレンジ `#CA5010` / 赤 `#D13438` / ティール `#038387`

### 3.2 タイポグラフィ

- フォント：`Segoe UI, Yu Gothic UI, sans-serif`（Windows 標準）
- ベース 14px、時刻軸ラベル 12px、ツールバー期間表示 20px semibold

### 3.3 エントリブロックのスタイル

- 背景：カテゴリ色の淡色（例：カテゴリ色 15% 不透明度）
- 左端：カテゴリ色の 4px バー
- 角丸 4px、ホバーで薄いシャドウ
- 選択中：カテゴリ色のボーダー 2px

## 4. ナビゲーションレール（画面切替）

ウィンドウ最左端に幅 48px の縦型アイコンバーを常設する。

```
┌──┬──────────────────────────────┐
│📅│ ← カレンダー（Phase 1・既定）    │
│☑ │ ← タスク（Phase 2）             │
│📊│ ← 分析（Phase 3）               │
│📓│ ← ノート（Phase 4）             │
│  │                               │
│⚙ │ ← 設定（下端固定）              │
└──┴──────────────────────────────┘
```

- 選択中のアイコンはアクセント色の背景 + 左端に 3px のアクセントバー
- 未実装フェーズのアイコンは Phase 1 では非活性（グレー + ツールチップ「近日対応」）
- 各画面はレール以外の領域（サイドパネル + コンテンツ）を丸ごと入れ替える

## 5. 設定画面

タブ構成のダイアログ：

1. **カテゴリ管理** — 一覧、追加、名前/色編集、アーカイブ
2. **表示** — 週の開始曜日（月/日）、表示時間帯（既定 7:00-22:00 にスクロール）、時刻刻み
3. **データ** — バックアップ先フォルダの指定（OneDrive 推奨・自動検出）、今すぐバックアップ、バックアップから復元（.db 選択 → 確認 → アプリ再起動で適用。旧データは pre-restore ファイルとして退避）、エクスポート（JSON / CSV）

補足（チケット画面の表示モード）:

- **リスト** — チケットの階層リスト（見積・実績のロールアップ、インライン編集）
- **カンバン** — Notion 風の 4 列ボード（To-Do / 進行中 / レビュー中 / 完了）。
  カードのドラッグでステータス変更。子を持つチケットは入れ物のため表示せず、
  作業単位（子タスク・子なしチケット）をカードにする
- **ガント** — Jira 風のタイムライン。チケット行のキャレットで子タスクを展開。
  バーは開始日〜期限日（未設定時は実績の記録日範囲から薄いバーで導出）。
  日付は行内の入力欄で設定、今日の位置に赤線

補足（カレンダーの表示モード）:

- **日表示** — 週グリッドと同じ時間軸で 1 列のみ（列幅 480px）。ドラッグ記入も同様に可能
- **月表示** — 6 週 × 7 日の概観グリッド。各セルに最大 3 件のエントリ（時刻 + タイトル）と「+n 件」を表示。セルをクリックするとその日の日表示へ移動

## 6. 将来の拡張とレイアウトの関係（参考）

- Phase 2（タスク）：左＝タスクリスト（見積時間付き）、右＝ミニ週カレンダー。タスクをカレンダーへドラッグして実績化
- Phase 2（検索）：ツールバー右上に検索ボックス（`Ctrl+F`）。入力でコンテンツ領域が結果リストに切り替わり、項目クリックで該当週へジャンプ + ハイライト（仕様詳細は requirements.md 9.1）
- Phase 3（分析）：期間セレクタ + カテゴリ別集計、見積 vs 実績、年間ヒートマップ（GitHub の草スタイル）
- Phase 4（ノート）：左＝フォルダツリー、右＝Markdown エディタ/プレビュー

ナビゲーションレール + 「サイドパネル + コンテンツ」の 2 段構造は、全画面がこの型に収まるように設計している。

## 7. チケット画面の改訂（2026-07-04）

チケット画面は「チケット / カンバン / ガント / 履歴」の4タブ構成に再編し、共通ヘッダー（タイトル + タブ切替）はどのタブでも同じ位置に固定した。

- **左レール（分類）**：スケジュールのカテゴリとは独立した軸。既定は「自学習」「プロジェクト」。
  「すべて」+ 各分類 + 「+ 分類を追加」。ダブルクリックで名前変更、ホバー時の × で削除（要確認）。
  ヘッダー左端のボタンでレール自体の表示/非表示を切替できる。
- **チケット**（新設）：Jira/Atlassian 風のマスタ・ディテール画面。左に親子階層のリスト
  （ステータスは角丸 4px の「ロゼンジ」バッジ）、右に選択項目の詳細パネル
  （ステータス・分類・カテゴリ・開始日/期限日・見積・実績合計・説明メモ・子タスク管理）。
  チケットの新規作成・削除はここが起点。
- **カンバン**：列は既定で To-Do / 進行中 / 完了 の3列（レビュー中はカードがある時のみ出現）。
  列ヘッダは中央寄せ。カードは**子タスクのみ**（親チケット自体は載せない）。
  完了列は当月に完了したものだけを表示し、月が変わると自然に消える。
  カード追加の UI はここには置かず、チケットタブから追加する。
- **ガント**：親チケット + 子タスクを表示（従来通り）。1 日 = 1 マス（30px 正方形）の
  日付ヘッダー行を追加し、バーとマスを整列。開始位置は既定で今日の3日前
  （設定 > 表示 > ガントチャートで日数を変更可）。タイムライン上のドラッグで
  開始日〜期限日を設定できる。左ペイン（チケット一覧）は右端をドラッグして幅を調整でき、
  最小幅は設定から変更できる。
- **履歴**（新設）：完了したチケット/タスクを完了日の年→月でグルーピングして振り返るビュー。
  年・月ごとに件数と実績合計時間を表示し、月をクリックすると項目一覧を展開できる。

削除操作は全画面共通でアプリ内モーダルの確認ダイアログ（`ConfirmDialog` / `confirmStore`）を経由する。
`window.confirm` は環境によって表示されないことがあるため使用しない。

## 8. ノート画面の改訂（2026-07-04）

Windows のファイルエクスプローラーに近い操作感に変更した。

- **新規作成**：サイドバーの「+」でツリーに直接「新しいノート」/「新しいフォルダ」が
  編集可能な状態で追加される（ポップアップは使わない）。Enter/フォーカスアウトで確定、Escape で取消。
- **名前変更**：既に選択中の項目をもう一度クリックするとインライン編集になる（ファイル・フォルダ共通）。
- **削除・名前変更**：ツリー項目を右クリックするとコンテキストメニュー
  （「名前を変更」「削除」）が出る。削除は確認ダイアログを経由する。
- エディタ右上の「名前変更」「削除」ボタンは廃止した（上記のツリー操作に統一）。
- **既定表示モード**：既存ノートを開くと**プレビュー**、新規作成したノートは**編集**が既定。

## 9. カレンダーのカテゴリ表示（2026-07-04）

サイドパネルのカテゴリ一覧はチェックボックス付きリストから、チップ（角丸ボタン）を
折り返し配置する表示に変更した。クリックで表示/非表示をトグルし、非表示中は淡色表示になる。
